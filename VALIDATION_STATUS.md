# Validation Status

TIPICA Webapp is a beta browser-based companion implementation. The archived Python desktop implementation remains the reference implementation for the manuscript results, and the webapp was not used to generate the submitted manuscript results.

| Area | Current status | Notes |
| --- | --- | --- |
| File list parity | Implemented | Implemented for the current Python-style reference package structure with `RESULTS/` and `RAW_DATA_DETAILS/`. |
| PNG dimensions | Implemented | Implemented for the current reference dimensions used by the browser export. |
| XLSX sheet order | Implemented | The report and diagnostics workbooks use the intended Python-style sheet order. |
| RESULTS report workbook content | Implemented | `comparison_after_36X_D3_method_comparison.md` shows `RESULTS/..._REPORT.xlsx` sheet order, sheet shapes, headers, structural text, and audited numeric fields matching the Python reference ZIP for the validated D3 reference pair. |
| RAW_DATA_DETAILS diagnostics workbook content | Implemented | `comparison_after_36X_D3_method_comparison.md` shows `RAW_DATA_DETAILS/..._DIAGNOSTICS.xlsx` sheet order, sheet shapes, headers, structural text, and audited numeric fields matching the Python reference ZIP for the validated D3 reference pair. |
| RGB pseudo-absorbance numerical parity | Under validation | RGB/PAbs values are being compared against the archived Python desktop output. |
| Fitting input parity | Under validation | Replicate aggregation and `n_points` semantics require continued parity checks. |
| Standard-addition C0 marker semantics | Partially implemented | Corrected and considered aligned with Python semantics after 36S, but still subject to broader parity validation. |
| CIELAB/DeltaE diagnostics | Partially implemented | Browser exports CIELAB/DeltaE diagnostics, but numerical and method-row parity remain under validation. |
| Method comparison | Partially implemented | Method-comparison outputs exist; row coverage and scoring parity are still under validation. |
| Background sampling | Under validation | Background-cell sampling, accepted-mask statistics, centroids, and RGB medians need parity review. |
| ROI/core/used-pixel diagnostics | Under validation | Robust ROI pixel-count and distribution-statistic parity remain under validation. |
| Geometry diagnostics | Under validation | Floor/mouth geometry source, radius, and well-bottom diagnostics require continued comparison with Python output. |
| Captions | Partially implemented | For the validated D3 reference pair, `RESULTS/..._RESULTS_CAPTION.txt` matches the Python reference ZIP. `RAW_DATA_DETAILS/..._RAW_DATA_DETAILS_CAPTION.txt` remains an intentional beta difference only in the `FIGURE_CIELAB_DELTAE` paragraph: the web text is kept because the current web artifact is now a Python-style composite fitting/report figure, and reverting to the Python ZIP wording would describe it less accurately as diagnostic CIELAB/DeltaE plots. All other RAW caption paragraphs match, so no caption-generator patch is currently needed for that validated pair. |
| Figure visual parity | Under validation | Figures use the Python-style output package names and dimensions; data and formatting parity remain under validation. |
| Manuscript result generation | Not used for manuscript results | Submitted manuscript results were generated by the archived Python desktop implementation, not by the webapp. |

The current webapp should be treated as a beta companion interface while this validation work continues.
